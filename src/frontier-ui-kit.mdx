# Frontier UI-Kit

The Frontier UI-Kit allows you to define the *look and feel* of your forms and use in all your application, with minimal effort.

This is a great opportunity to bring a cohesive experience to your users across all the forms of your application.

<br />

## Choose your rendering mode

If render props are provided, `<Frontier>` will set-up the [`kit` render props argument](#kit-render-props-argument) using the provided UI Kit.

If no render props are provided, `<Frontier>` will render the form using the *full rendering*  feature.

<br />
<br />

### Full-rendering

Just providing an `uiKit=` props to `<Frontier>` will trigger a *full rendering* from Frontier.

`<Frontier>` will call the provided `UIKitAPI.form()` in order to wrap all the fields.

Then, for each field of the mutation, it will apply the following process:

Example, for a field "email" of type `String`:

1. Frontier will first try to get a registred `path()` component for "email"
2. Then, Frontier will try to get a registred `type()` component for the type `String`
3. Finally, Frontier will fallback to the `Unknown component`

### Render-props

Frontier UI-kit is also compatible with render-props.

By using the [`kit` render props argument](#kit-render-props-argument), you have the full control on the final rendering.
